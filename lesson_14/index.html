CSS переходи (CSS transition)

CSS - переходи або CSS - transitions можуть застосовуватися до всіх елементів і навіть псевдоелементів. Використовується для оживлення нашого макета. Що, в свою чергу, призводить до вдосконалення взаємодії з користувачем, як правило, завдяки приємній анімованій реакції на його дії. Наприклад - наведення та натиснення на кнопку.
Також властивості переходу можна використовувати для створення простих сценаріїв анімації. Насправді, CSS - переходи забезпечують зміну значень інших властивостей із певною анімацією та сценарієм.
Вся ця магія може бути описана універсальною CSS властивістю transition або наступними окремими властивостями:

Transition-duration

Визначає період часу, протягом якого слід здійснювати перехід. Власне - тут ми вказуємо час, за який одне значення властивості переходить (анімується) до іншого. Властивість не успадковується.

:0s

button

:0.4s

button

Transition-property

Містить назву CSS-властивості, до якої буде застосований ефект переходу. Значення властивості може містити одну властивість так і перелік властивостей через кому. Власність не успадковується.

:all

button

:padding

button

:padding, border-radius

button

Transition-delay

Затримка виконання переходу. Дозволяє зробити так, що зміна властивості відбудеться не миттєво, а з певною затримкою. Час затримки переходу вказується в секундах або мілісекундах. Властивість не успадковується.

:0s

button

:1s

button

Transition-timing-function

Встановлює стиль нашого переходу, тобто певний сценарій, згідно з якою буде проведена анімація.

:ease

button

:linear

button

:steps(3, end)

button

:cubic-bezier(0.55, -0.22, 0.77, -0.2)

button

Універсальний запис

Для елемента можна встановити кілька послідовних переходів, перелічивши їх через кому.

:___,___,___

button

CSS анімація (CSS animation & @keyframes)

Як і властивість CSS плавних переходів transition, властивість анімації розроблена для того, щоб зробити наш макет більш динамічним, кращим для взаємодії з користувачем та створити ефект WOW.
Але на відміну від переходів CSS, створення анімації базується на ключових кадрах @keyframes, які дозволяють автоматично відтворювати та повторювати ефекти за заданий час, а також призупинити анімацію за певної події.
Іншими словами, використання анімації та @keyframes дозволяє нам створювати більш складні сценарії анімації.

Animation-name

Визначає перелік елементів анімації (ключових кадрів). Можна вказати через кому. Пріоритет у останнього запису.

animation-name: pulse;

@keyframes pulse {
0%{}
100%{}
}

animation-duration

Відповідає за тривалість анімації.

:1s

:2s

Animation-timing-function

Встановлює сценарій для програвання анімації між ключовим кадрами.

:ease

:ease-in-out

:steps(3,end)

:cubic-bezier
(0.6,-0.3,0.8,-0.2)

Animation-iteration-count

Відповідає за повторення відтворення ключових кадрів.

:1

:3

:infinite

animation-direction

Визначає напрямок та тип відтворення ключових кадрів.

:normal

:reverse

:alternate

:alternate-reverse

Animation-play-state

Дозволяє запускати або зупиняти анімацію за подією.

:running

:paused :hover

:running :hover

Animation-delay

Відповідає за затримку перед відтворенням.

:0s

:3s

Animation-fill-mode

Визначає, які значення анімованих властивостей CSS застосовуються до об'єкта після завершення анімації.

:none

:forwards

:backwards

:both

Універсальний запис

Для елемента можна встановити кілька анімацій, перелічивши їх через кому.

:__ __ __, __ __ __
